running svc-json-mv-indexDB-OPT-04 v1.0.0 2,410 421d 09h ▾

MySQL 8.0.17+ multi-value index over a nested JSON array column, migrating a high-traffic hot query from full JSON scan to direct B-tree lookup.

situationa high-QPS hot query filtered rows by membership in a JSON-array column. Every call triggered a full column scan + JSON parse across millions of rows. p99 latency climbed, CPU saturated nightly, false-positive pager alerts followed.
taskbring the hot path under an index without denormalising the JSON column, splitting the table, or rewriting upstream callers.
actionadded a MySQL 8.0.17+ multi-value index over the array path · rewrote the predicate from JSON_CONTAINS to MEMBER OF · verified EXPLAIN showed index use · rolled out behind a feature flag with shadow-traffic comparison.
result~15% reduction in prod CPU on the affected stack · p99 on the hot query dropped from ~740ms → ~11ms · 20–30 false-positive pager alerts/week eliminated.
